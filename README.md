# Реализация паттерна Observer (Наблюдатель) на C++

Этот проект демонстрирует работу поведенческого паттерна проектирования **Observer**, который позволяет объектам подписываться на уведомления о событиях в другом объекте.

## Основная концепция

Класс `Store` (Издатель) содержит список подписчиков (`Customers`). Когда в магазине меняется состояние (устанавливается новая скидка), он автоматически оповещает всех зарегистрированных клиентов.

### Преимущества данного подхода:
1.  **Слабая связь (Loose Coupling):** Магазин не знает деталей реализации конкретных клиентов, он взаимодействует с ними через абстрактный интерфейс `Customer`.
2.  **Гибкость:** Новые типы уведомлений (SMS, Push-уведомления) можно добавить без изменения кода класса `Store`.
3.  **Динамичность:** Подписка и отписка происходят в процессе выполнения программы (`runtime`).

---

## Механизмы работы

### 1. Подписка и управление (Subscription)
Управление списком наблюдателей реализовано через методы:
*   `subscribe(Customer*)`: Добавляет указатель на объект клиента в вектор `Customers`.
*   `unsubscribe(Customer*)`: Находит клиента в векторе и удаляет его, прекращая рассылку.

### 2. Уведомление (Notification)
Метод `notify()` проходит циклом по всем элементам вектора и вызывает у каждого метод `update()`. Это гарантирует, что каждый подписчик получит актуальную информацию мгновенно после вызова `setDiscount()`.

---

## Варианты получения данных: Push vs Pull

В представленном коде реализованы обе модели передачи данных:

### Модель Push
Данные передаются напрямую в аргументах метода `update`.
*   **Как это работает:** Объект `Store` сам решает, какую информацию отправить (сообщение и процент скидки).
*   **Пример в коде:** Классы `RegelarCustomer` и `VIPCustomer`.
*   **Плюс:** Клиенту не нужно запрашивать детали, он сразу их получает.

### Модель Pull
Клиент получает ссылку/указатель на объект источника и сам запрашивает нужные ему данные.
*   **Как это работает:** В методе `update` используется указатель `Store* store`, через который вызывается метод `store->getCurrentDiscount()`.
*   **Пример в коде:** Класс `EmailCustomer`.
*   **Плюс:** Клиент сам выбирает, какие именно данные из состояния магазина ему важны в данный момент.
